# 待改进清单（Backlog）

## 说明

本文档用于记录“智能仓储协同系统”的后续优化项。

当前阶段策略：**Demo 优先**，先保证核心功能完整可演示；工程化与平台化能力后置。

## 已完成（本轮）

1. WebSocket 心跳机制（前后端）
2. WebSocket 断线自动重连（前端指数退避）

## P0（建议尽快）

1. 增加数据库集成测试（并发扣减、防超卖、幂等）
   - 目标：在真实 PostgreSQL 环境验证 `SELECT FOR UPDATE` 语义与业务键幂等。
   - 覆盖：并发 50~200 请求、重复 `order_id` 重放、库存边界值。

2. 完善鉴权与安全
   - 增加登录限流与失败锁定策略。
   - JWT 增加签发方与受众校验。
   - 管理员密码改为带盐哈希（如 Argon2/Bcrypt）。

3. 请求与事件可观测性（Demo 最小版）
   - 增加请求 ID、结构化日志、关键操作审计追踪。
   - 记录库存变更链路耗时（API → DB → WS 广播）。

3. 前端交互增强
   - 增加库存变更明细抽屉（基于 `inventory_logs`）。
   - 增加补货建议筛选与状态更新。
